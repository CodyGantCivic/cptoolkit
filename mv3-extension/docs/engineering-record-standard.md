# Engineering Record Standard

This project should not depend on chat history as the only record of important work.

For architecture, release readiness, Chrome Web Store review, security hardening, permissions, or multi-step implementation plans:

1. Create or update a repo-local note under `docs/`.
2. Record current state, decisions, open questions, and next actions.
3. Link relevant task ids, PRs, source docs, or external policy references.
4. Keep the note current when the plan changes in the same branch.
5. Mention the note in the final handoff so the next developer knows where to continue.
