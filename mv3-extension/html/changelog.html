Version 1.2.0

New Tools, Snippets Page & Options Enhancements
New Features
  • CSS Snippet Library — Full Page Viewer: New dedicated page for viewing, editing, creating, and deleting CSS snippets. Accessible via the expand button in the sidebar snippet panel. Features grid and list views, category grouping with icons, inline code blocks with syntax-themed dark backgrounds, and copy-to-clipboard on each snippet.
  • Copy Multiple Skins: New on-demand tool for bulk copying widget skins between sites. Export tab saves skins from the current site, Import tab applies saved skins to targets on another site. Supports auto-match by name, creating new skins, and custom naming. Two-phase save process for new skin creation (create → save → apply styles → save).
  • On-Demand Tools Options: Added a new "On-Demand Tools (Context Menu)" section to the Options page. Users can now disable individual on-demand tools to hide them from the right-click context menu. Changes take effect immediately.
Enhancements
  • CSS Snippets — Interaction Model: Changed quick snippets dropdown to hover-to-open (previously click). Click now opens the full sidebar panel. Removed hardcoded built-in snippets — dropdown now only shows user-saved snippets marked as "always in quick list".
  • CSS Snippets — Generic Storage Bridge: Added a reusable storage bridge for MAIN world scripts to read/write chrome.storage.local via CustomEvents. Used by the Copy Multiple Skins tool and available for future on-demand tools.
  • CSS Snippets — Fullscreen Button: Added expand icon next to the sidebar close button that opens the full-page snippet library in a new tab.
Bug Fixes
  • CSS Snippets Dropdown: Fixed gray border appearing at top of empty quick snippets dropdown. Made separator conditional on having items above it.
  • CSS Snippets Dropdown: Fixed empty dropdown still showing a bordered box when visible but containing no items.
  • Mini IDE: Fixed empty dropdown showing border with CSS rule to hide .css-code-popup when visible but has no list items.
  • Copy Multiple Skins: Fixed "Widget skin to be modified does not exist on Db" error when creating new skins. Now uses a two-phase approach: create and save first, then apply styles and save again.
  • Copy Multiple Skins: Fixed processNewSkin opening the CMS Manage Widget Skins modal during bulk operations. Intercepts the function to perform only data operations without UI refresh.
  • Snippets Page: Fixed skins.forEach is not a function error when copied skins were stored as an object instead of an array.
UI Updates
  • CSS Snippets Dropdown: Removed "User" badge from dropdown items since hardcoded snippets were removed.
  • Snippet Library Page: Grid view displays exactly 2 columns. Lone items on the last row or in single-item categories expand to full width. Code blocks wrap long lines instead of horizontal scrolling.

Version 1.1.0

New Features, Bug Fixes & UI Updates
New Features
  • Import Fancy Buttons — Social Icons Tab: Bulk-upload social media icons (Black, White, Color, or All variants) as graphic links. Select icons, choose a Document Center folder, and upload them all at once with a progress bar and log.
  • Import Fancy Buttons — Export JSON: Export any existing fancy button's styles as JSON. Options to copy to clipboard, download as a file, or save to your personal button library.
  • Import Fancy Buttons — Custom Button Library: Save exported buttons for reuse. Your saved buttons appear alongside built-in templates in the Template Library tab and can be deleted individually.
  • Prevent Session Timeout — Server Keep-Alive: Sends a background ping every 5 minutes to keep the ASP.NET server session alive, preventing save failures after long idle periods.
  • Prevent Session Timeout — Badge Counter: The extension icon now shows a badge count of how many times the session was auto-refreshed.
  • Link Autofill — New Socials: Added autofill entries for Bluesky, Everbridge, Flickr, Mastodon, Nextdoor, RSS, Threads, and TikTok.
Bug Fixes
  • Widget Skin Default Override: Fixed the Manage Widget Skins modal opening twice after creating or copying a skin. The tool was explicitly reopening the modal, but the CMS save callback already does that automatically.
  • Enforce Advanced Styles Text Limits: Fixed MiscAdvStyles maxlength from 255 to 1000 to match the actual server-side limit for widget skin advanced styles.
  • Export Fancy Button: Fixed export not capturing advanced styles. Textarea fields (where advanced styles live) were not being collected — only inputs and selects were.
  • Mini IDE: Fixed errors when MutationObservers tried to attach before document.body existed.
  • Remember Image Picker State: Fixed the tool from interfering with hidden iframes created by other toolkit tools (folder lookup, upload flows).
  • Prevent Session Timeout: Rewrote to directly detect the Session Timeout popover modal and dismiss it via MAIN world execution, instead of relying on the older inline UI message pattern. Reduced check interval from 2 minutes to 15 seconds.
UI Updates
  • Import Theme: Redesigned from a bare file picker to a full styled modal matching the toolkit style guide.
  • Multiple Category Upload: Restyled modal to match the toolkit style guide with proper header/body/footer layout and CivicPlus red buttons.
  • Import Fancy Buttons: Updated modal styling — CivicPlus red for active tabs, primary buttons, and focus rings. Fixed button line-height conflicts with CMS styles.
